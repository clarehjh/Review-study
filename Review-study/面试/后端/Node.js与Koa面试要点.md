##### （1）Koa 中间件洋葱模型（面试必问）

- 核心讲解：
  “Koa 的中间件是按‘洋葱模型’执行的，比如我在羽球小程序后端中，先写了日志中间件，再写登录鉴权中间件，最后写业务路由中间件。请求进来时，先执行日志中间件的外层代码→鉴权中间件外层代码→路由处理，响应返回时，再反向执行鉴权中间件内层代码→日志中间件内层代码。这样的设计可以让中间件复用性更高，比如所有接口都能通过日志中间件记录请求信息。”
- 简单示例（不用手写完整代码，讲清逻辑即可）：
  javascript

  运行

  ```
  const Koa = require('koa');
  const app = new Koa();

  // 日志中间件（洋葱外层）
  app.use(async (ctx, next) => {
    console.log('请求开始：', ctx.url);
    await next(); // 执行下一个中间件
    console.log('请求结束：', ctx.status); // 洋葱内层
  });

  // 业务路由中间件
  app.use(async (ctx) => {
    ctx.body = { code: 200, data: '计分数据' };
  });

  app.listen(3000);
  ```

##### （2）小程序登录鉴权（你的项目核心，必讲）

- 核心讲解（结合你的项目）：
  “我的羽球小程序登录流程是：前端获取微信小程序的 code→把 code 传给 Koa 后端→后端调用微信官方接口（auth.code2Session）换取 openid 和 session_key→用 JWT 生成 token 返回给前端→前端存储 token，后续请求接口时携带 token→后端通过 JWT 中间件验证 token 合法性，这样既保证了用户唯一标识，又避免了存储 session 的开销。”
- 关键代码片段（面试时能写出核心逻辑即可）：
  javascript

  运行

  ```
  const jwt = require('jsonwebtoken');
  const axios = require('axios');

  // 登录接口
  router.post('/login', async (ctx) => {
    const { code } = ctx.request.body;
    // 1. 调用微信接口换openid
    const res = await axios.get('https://api.weixin.qq.com/sns/jscode2session', {
      params: {
        appid: '你的小程序appid',
        secret: '你的小程序secret',
        js_code: code,
        grant_type: 'authorization_code'
      }
    });
    const { openid, session_key } = res.data;
    // 2. 生成JWT token
    const token = jwt.sign({ openid }, '你的秘钥', { expiresIn: '7d' });
    // 3. 返回给前端
    ctx.body = { code: 200, token, openid };
  });

  // 鉴权中间件
  app.use(async (ctx, next) => {
    if (ctx.url.includes('/api')) { // 接口需要鉴权
      const token = ctx.headers.authorization?.split(' ')[1];
      if (!token) {
        ctx.body = { code: 401, msg: '未登录' };
        return;
      }
      // 验证token
      try {
        jwt.verify(token, '你的秘钥');
        await next();
      } catch (e) {
        ctx.body = { code: 401, msg: 'token过期' };
      }
    } else {
      await next();
    }
  });
  ```

##### （3）接口跨域处理（前端 + Node 都可能问）

- 核心讲解：
  “小程序本身不限制跨域，但我在开发时用 H5 调试小程序前端，遇到了跨域问题。在 Koa 后端中，我用了 koa2-cors 中间件，配置允许的源、请求方法和头信息，解决了跨域问题。”
- 代码示例：
  javascript

  运行

  ```
  const cors = require('koa2-cors');
  app.use(cors({
    origin: '*', // 生产环境改为指定域名
    allowMethods: ['GET', 'POST', 'PUT', 'DELETE'],
    allowHeaders: ['Content-Type', 'Authorization']
  }));
  ```

### 三、补充复习建议

1. **不用深钻 Node.js**：你的核心还是前端，Node.js 只需要聚焦 “你项目中用到的知识点”（Koa 中间件、小程序登录、接口开发），不用去学 Express、Nest.js 等其他框架，避免分散精力；
2. **梳理 “问题 + 解决方案”**：回忆你开发后端时遇到的问题（比如接口超时、数据存储报错、token 过期），想好对应的解决方案，面试时讲出来，比单纯说 “会 Node” 更有说服力；
3. **结合工程化**：如果你的 Node 后端也做了自动化部署（比如用 Jenkins 部署 Node 服务），可以把这部分和前端自动化部署结合起来讲，突出 “全流程自动化” 的能力。
