## 第一组：停止、撤回、分叉

> 当 Claude Code 正在做事，但你想打断、后退，或者换一条路继续走时，怎么办？

### 1. `Esc` — 先学会停

停止当前任务。**重要**：停的是正在进行的动作，不是整段会话。聊天历史和上下文都在，只是眼前这一步不再继续跑。

> 跑偏、卡住时先按 Esc，比另起炉灶省事得多。

### 2. 双击 `Esc` 或 `/rewind` — 回到上一步

不是简单中断，而是撤销菜单。通常会给你几个方向：

- 恢复代码和对话
- 只恢复对话
- 只恢复代码
- 从这里开始做总结
- 取消

> 不必在"全盘接受"和"全部重来"之间二选一。很多时候你只想回退代码，不想丢掉分析过程。

### 3. `/btw` — 把岔出去的小问题留在旁边

处理"我突然想问一句，但不想打断主线"的情况：

```bash
/btw where is the test file again
```

> 给旁支问题留一个出口，不污染当前任务。

### 4. `/branch` — 把思路真的分成两条

一个任务有两个方案，不确定哪条更好？直接把当前会话分叉，两边都试，保留跑通的那个版本。

> 类似 git 分支，但分的是对话和推理过程，不是代码仓库。

## 第二组：控制上下文窗口

> 会话怎么继续长，但上下文不要一起膨胀。

### 5. `/compact` — 把长历史压成还能继续工作的摘要

不是简单删历史，而是把长对话重写为更紧凑的摘要，同时尽量保留主线、技术决策、报错和修复过程。

> 你需要的不是清掉历史，而是把历史压缩成还能接着工作的形态。常聊一小时以上的任务，这个命令几乎应该变成习惯。

### 6. `/clear` — 彻底换题时，比硬切更干净

清空当前上下文，开始新话题。适合的不是"我有点乱"，而是"我已经完全切题了"。

> 明明要聊两个完全不同的问题，却还在同一个会话里硬续——上下文里混进了太多无关信息，/clear 比继续补 prompt 更有效。

### 7. `/export` — 把一次长会话沉淀成文件

导出为 Markdown，路径如：`~/projects/XXX/claude-session-YYYY-MM-DD-HH-MM.md`

> 花一个小时把系统设计思路讲清楚，不导出来，下次再找就很麻烦。对话本身就是工作记录。

### 8. `/resume` — 靠关键词把旧会话捞回来

按关键词找旧会话，而不是翻历史列表硬找。

> 你真正缺的往往不是最近一次，而是"上周那个聊过支付回滚策略的会话"。

### 9. `claude -c` — 继续当前目录里最近的一次会话

CLI 层面的续接方式。昨天在这个目录里做到一半，今天回来接着做，不用重新描述上下文。

### 10. `claude -r` — 把历史会话当成可跳转的工作记录

列出过去的会话，让你跳回指定 session。

| 命令 | 场景 |
|------|------|
| `claude -c` | 接着上次继续 |
| `claude -r` | 回到某一次具体讨论 |

### 11. `/remote-control` (`/rc`) — 把会话带走，执行还留在你的机器上

让正在本机运行的会话，交给手机等远端设备来操控。执行还在你的机器上，你只是换了一个操纵入口。

> 代码和任务在电脑上跑着，你暂时离开工位，但还想看进度、补一句话、做个确认。

## 第三组：不是更努力，而是更会调用能力

> 让你更方便地调模型、调视角、调自动化。

### 12. `/model opusplan` — 把规划和执行拆开

规划用更慢、更强的模型，执行用更快的模型。适合架构设计、复杂重构、跨文件修改前的方案分析。

> 很多人默认从头到尾只用一个模型，其实容易把"该慢想的地方"和"该快干的地方"混在一起。

### 13. `/simplify` — 一口气拉三个 reviewer 看同一份东西

并行拉出三个 reviewer，分别看：

| Reviewer | 关注点 |
|----------|--------|
| 1 | 架构和复用 |
| 2 | 代码质量 |
| 3 | 效率 |

> 拿到的不是一段大而化之的反馈，而是从三个不同角度拆过一遍的结论。

### 14. `/insights` — 把自己的使用习惯也当成可以优化的对象

在本地生成 HTML 报告（`~/.claude/usage-data/report.html`），分析你的常见习惯、反复犯的错误、没用过的功能，以及对 CLAUDE.md 的具体建议。

> 很多人会反复研究 prompt，却不怎么复盘自己的使用方式。你的工作流本身也值得被分析。

### 15. `/loop` — 把提醒和巡检变成会话内的定时任务

```bash
/loop 15m check the deploy
/loop in 20m remind me to push this branch
```

可以放一个 `.claude/loop.md` 定义默认行为，它就不只是提醒器，而是一个轻量的周期任务入口。

### 16. `/skills [name]` — 把项目能力按工作区装进去

| 用法 | 效果 |
|------|------|
| `/skills` | 查看当前工作区有哪些可用技能 |
| `/skills <name>` | 加载指定 skill |

> 真正有复利的地方，不只是单次问答，而是你能不能把项目知识变成可重复调用的工作流。

## 第四组：高频操作，别每次都手动做

| 快捷键 | 作用 | 场景 |
|--------|------|------|
| `Ctrl+V` | 直接粘贴截图 | 报错弹窗、UI 问题、设计图、终端截图，不用先保存再上传 |
| `Ctrl+J` / `Option+Enter` | 换行但不发送 | 写多行 prompt 时防止手滑直接发出 |
| `Ctrl+R` | 搜索 prompt 历史 | 高质量提问是重复模式，搜出来改比重写快 |
| `Ctrl+U` | 清空整行输入 | 写了一长串发现方向不对，一键清掉 |

## 🎯 如果只学 5 个，先学这些

| 优先级 | 命令 | 学会什么 |
|--------|------|----------|
| 1 | `Esc` | 先学会中断 |
| 2 | `/rewind` | 学会回退而不是重开 |
| 3 | `/compact` | 学会上下文压缩 |
| 4 | `/resume` 或 `claude -c` | 学会把会话接起来 |
| 5 | `/loop` | 学会把等待类任务交给系统 |

> 这 5 个加起来，已经足够把 Claude Code 从"终端里的聊天工具"变成"有连续性的工作流工具"。之后再慢慢加 `/branch`、`/simplify`、`/insights`、`/skills`——上限不在单次问答，而在会话管理和工作流组织。
