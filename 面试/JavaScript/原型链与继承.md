# 一、原型链 / 原型继承（手写继承、new 关键字实现）面试答题思路（先理解核心概念，再手写代码）

#### 1. 先回答原型链 / 原型继承的核心概念（面试开场铺垫）

- **原型链**：JavaScript 中每个对象都有 `__proto__` 属性（隐式原型），指向它的构造函数的 `prototype`（显式原型）；当访问一个对象的属性 / 方法时，若自身没有，会沿着 `__proto__` 向上查找，直到 `Object.prototype.__proto__`（值为 `null`），这条查找链就是原型链。
- **原型继承**：JavaScript 没有类（ES6 `class` 是语法糖），继承本质是**通过原型链实现属性 / 方法的复用**，核心是让子类的原型指向父类的实例，从而能访问父类原型上的属性和方法。
- **原型链简单记忆：每个产品（实例）都有个二维码（proto），扫一下就能找到工厂的模板库（prototype）；模板库里的工具所有产品都能用，不用每个产品都造一遍。原型链的本质就是：对象找属性 / 方法时的 “查找链条” —— 先找自己的，找不到就扫二维码去原型上找，还找不到就去原型的原型上找，直到找到或到顶（null）。**

#### 2. 手写 `new` 关键字的实现（面试高频手写题）

`new` 关键字执行时会做 4 件事：

1. 创建一个空的新对象；
2. 让新对象的 `__proto__` 指向构造函数的 `prototype`；
3. 执行构造函数，将 `this` 绑定到新对象上，传递参数；
4. 判断构造函数的返回值：若返回引用类型（对象 / 函数），则返回该值；否则返回新创建的对象。

#### 3. 手写原型继承（面试常考的几种方式）

```


// 模拟实现new关键字
function myNew(constructor, ...args) {
  // 1. 创建空对象，让其__proto__指向构造函数的prototype
  const obj = Object.create(constructor.prototype);
  // 2. 执行构造函数，绑定this到新对象，传递参数
  const result = constructor.apply(obj, args);
  // 3. 判断返回值：若为引用类型则返回，否则返回新对象
  return typeof result === 'object' && result !== null ? result : obj;
}

// 测试用例
function Person(name, age) {
  this.name = name;
  this.age = age;
  // 测试返回引用类型的情况（可选）
  // return { gender: 'male' };
}
//工厂的公共模板库Person.prototype
Person.prototype.sayHi = function() {
  console.log(`Hi, I'm ${this.name}`);

};

// 使用自定义myNew
const p1 = myNew(Person, '张三', 20); //实例
console.log(p1.name); // 张三
p1.sayHi(); // Hi, I'm 张三

function Person(name, age) {
  this.name = name;
  this.age = age;
  const res = { gender: "male" };
  res.__proto__ = Person.prototype; // 手动关联原型 p1.sayHi()就可以运行
  return res;
}
new（包括 myNew）的核心规则：如果构造函数返回非 null 的引用类型（对象 / 数组等），则返回这个值，否则返回关联了原型的新对象；
你加的 return { gender: "male" } 让 p1 变成了独立对象，和 Person.prototype 彻底断开关联；
只有当构造函数没有返回值（或返回基本类型，如 return 123/return 'abc'）时，myNew 才会返回关联了 Person.prototype 的 obj，此时 p1.sayHi() 才能正常执行。
```

##### 方式 1：基础原型继承（核心思路）

javascript

```
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['篮球', '游戏']; // 引用类型属性
}
Parent.prototype.sayName = function() {
  console.log(`My name is ${this.name}`);
};

// 子类
function Child(age) {
  this.age = age;
}

// 核心：让子类原型指向父类实例，实现继承
Child.prototype = new Parent('父类默认名');
// 修复构造函数指向（否则Child的实例.constructor会指向Parent）
Child.prototype.constructor = Child;

// 子类扩展方法
Child.prototype.sayAge = function() {
  console.log(`I'm ${this.age} years old`);
};

// 测试
const child1 = new Child(10);
child1.sayName(); // My name is 父类默认名
child1.sayAge(); // I'm 10 years old

// 注意：基础原型继承的问题——引用类型属性共享
const child2 = new Child(12);
child1.hobbies.push('看书');
console.log(child2.hobbies); // ['篮球', '游戏', '看书']（所有实例共享）
```

运行

##### 方式 2：组合继承（最常用，解决引用类型共享问题）

组合继承 = 构造函数继承（传参 + 解决引用类型共享） + 原型链继承（复用原型方法）

javascript

运行

```
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['篮球', '游戏'];
}
Parent.prototype.sayName = function() {
  console.log(`My name is ${this.name}`);
};

// 子类
function Child(name, age) {
  // 1. 构造函数继承：调用父类构造函数，绑定this，传递参数（解决引用类型共享+传参）
  Parent.call(this, name);//Parent.call(this) = “让子类实例‘借用’父类构造函数，把父类的属性都复制到自己身上”
  this.age = age;
}

// 2. 原型链继承：复用父类原型方法
Child.prototype = new Parent();
Child.prototype.constructor = Child;

// 子类扩展方法
Child.prototype.sayAge = function() {
  console.log(`I'm ${this.age} years old`);
};

// 测试
const child1 = new Child('张三', 10);
const child2 = new Child('李四', 12);

child1.hobbies.push('看书');
console.log(child1.hobbies); // ['篮球', '游戏', '看书']
console.log(child2.hobbies); // ['篮球', '游戏']（不再共享）

child1.sayName(); // My name is 张三
child2.sayAge(); // I'm 12 years old
```

##### 方式 3：寄生组合继承（最优解，解决组合继承重复调用父类构造函数的问题）

组合继承的问题：`Child.prototype = new Parent()` 会调用一次父类构造函数，`Parent.call(this)` 又调用一次，导致子类原型上有多余的属性；寄生组合继承可避免这个问题。

javascript

运行

```
// 父类
function Parent(name) {
  this.name = name;
  this.hobbies = ['篮球', '游戏'];
}
Parent.prototype.sayName = function() {
  console.log(`My name is ${this.name}`);
};

// 子类
function Child(name, age) {
  Parent.call(this, name); // 仅调用一次父类构造函数
  this.age = age;
}

// 核心：创建空对象，让其__proto__指向父类原型，再赋值给子类原型（避免重复调用父类构造）
Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child;

// 子类扩展方法
Child.prototype.sayAge = function() {
  console.log(`I'm ${this.age} years old`);
};

// 测试（效果和组合继承一致，但性能更好）
const child = new Child('王五', 15);
child.sayName(); // My name is 王五
child.sayAge(); // I'm 15 years old
```

### 二、面试答题技巧

1. **先讲概念**：回答原型链 / 继承时，先简单说清楚 “原型链是属性查找的链条，继承基于原型链实现”，再讲 `new` 的作用。
2. **手写代码时注释清晰**：面试手写代码时，加关键注释（比如 `// 修复构造函数指向`），体现你理解每一步的意义。
3. **讲优缺点**：写完继承代码后，主动说不同继承方式的优缺点（比如基础原型继承有引用类型共享问题，组合继承解决了但调用两次父类构造，寄生组合继承是最优解），加分项。

### 总结

1. `new` 关键字的核心：创建新对象 → 绑定原型 → 执行构造函数 → 返回新对象（或构造函数的返回值）。
2. 原型继承的核心：让子类原型指向父类相关对象（实例 / 父类原型），实现属性 / 方法复用；组合继承（或寄生组合继承）是面试中最推荐写的方式，能解决基础继承的问题。
3. 答题逻辑：先解释概念 → 手写核心代码（加注释） → 补充代码优缺点，体现对原型继承的深度理解。
