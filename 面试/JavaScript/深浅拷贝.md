# 前端面试之深浅拷贝（含高频问题 + 手写深拷贝）

## 一、面试标准回答框架（先总后分，逻辑清晰）

### 1. 核心区别（一句话讲透）

* **浅拷贝**：仅复制对象 / 数组的表层结构，若属性值为引用类型（对象、数组），拷贝后仍指向同一内存地址，修改拷贝后的引用类型属性会同步影响原数据；
* **深拷贝**：复制对象 / 数组的所有层级，所有引用类型属性都会创建新的内存空间，拷贝后与原数据完全隔离，互不影响。

### 2. 浅拷贝的常见实现方式

javascript

运行

```
// 数组浅拷贝
const arr = [1, { a: 2 }];
const arrCopy1 = [...arr];
const arrCopy2 = arr.slice();
const arrCopy3 = Array.from(arr);

// 对象浅拷贝
const obj = { x: 1, y: { z: 2 } };
const objCopy1 = { ...obj };
const objCopy2 = Object.assign({}, obj);

// 验证浅拷贝特性
arrCopy1[1].a = 3; 
console.log(arr[1].a); // 3（原数组同步修改）
objCopy1.y.z = 4;
console.log(obj.y.z); // 4（原对象同步修改）
```

### 3. 深拷贝的实现（面试核心）

#### （1）基础版（不考虑循环引用，面试入门写法）

javascript

运行

```
function deepCloneBasic(target) {
  // 非引用类型直接返回（终止递归）
  if (typeof target !== 'object' || target === null) {
    return target;
  }
  
  // 初始化拷贝结果（区分数组/对象）
  const result = Array.isArray(target) ? [] : {};
  
  // 遍历自身属性并递归拷贝
  for (let key in target) {
    if (target.hasOwnProperty(key)) {
      result[key] = deepCloneBasic(target[key]);
    }
  }
  
  return result;
}
```

`hasOwnProperty` 的核心作用是：**无论原型链上有没有可枚举属性，都能精准过滤掉，只拷贝对象 / 数组自己定义的属性**（比如数组的索引、对象直接定义的键值对），保证拷贝结果的纯净性，避免混入原型链的无关属性。

#### （2）完整版（处理循环引用，面试加分写法）

javascript

运行

```
function deepClone(target, map = new WeakMap()) {
  // 1. 非引用类型直接返回
  if (typeof target !== 'object' || target === null) {
    return target;
  }
  
  // 2. 处理循环引用：缓存已拷贝对象
  if (map.has(target)) {
    return map.get(target);
  }
  
  // 3. 初始化结果（区分数组/普通对象）
  const result = Array.isArray(target) ? [] : {};
  
  // 4. 缓存当前对象，避免循环引用
  map.set(target, result);
  
  // 5. 递归拷贝所有自身属性
  for (let key in target) {
    if (target.hasOwnProperty(key)) {
      result[key] = deepClone(target[key], map);
    }
  }
  
  return result;
}

// 测试循环引用
const obj = { a: 1 };
obj.b = obj; // 循环引用：obj.b指向自身
const objCopy = deepClone(obj);
console.log(objCopy.b === objCopy); // true（拷贝后循环引用正常）
console.log(objCopy.b === obj); // false（与原对象完全隔离）
```

#### （3）内置 API 实现深拷贝（说明优缺点）

javascript

运行

```
// JSON.parse(JSON.stringify())：简单但有缺陷
const obj = { a: 1, b: new Date(), c: function() {}, d: undefined };
const objCopy = JSON.parse(JSON.stringify(obj));
console.log(objCopy); // { a: 1, b: "2026-02-25T08:00:00.000Z" }
```

**缺陷**：

* 无法拷贝函数、undefined、Symbol（会被忽略）；
* 无法处理循环引用（直接报错）；
* 特殊类型（Date、RegExp、Map、Set）会丢失原类型；
* 丢失对象原型链。

## 二、面试高频问题（按提问概率排序）

### 1. 基础必问

#### （1）深浅拷贝的核心区别是什么？

答：核心是**是否复制引用类型的深层内存空间**。浅拷贝仅复制表层结构，引用类型共享内存；深拷贝复制所有层级，引用类型重新创建，拷贝后与原数据完全隔离。

#### （2）浅拷贝有哪些实现方式？

答：

* 数组：扩展运算符 `[...arr]`、`arr.slice()`、`Array.from(arr)`；
* 对象：扩展运算符 `{ ...obj }`、`Object.assign({}, obj)`；
* 注意：`Object.assign` 是浅拷贝，若目标对象有同名属性会覆盖。

#### （3）JSON.parse (JSON.stringify ()) 实现深拷贝的缺陷？

答：

1. 无法拷贝函数、undefined、Symbol 类型（会被忽略）；
2. 无法处理循环引用（执行时直接报错）；
3. Date 类型会被转为字符串，RegExp/Map/Set 等特殊类型会丢失原特性；
4. 拷贝后的对象丢失原型链，无法继承原对象的方法。

### 2. 手写代码必问

#### （1）手写深拷贝（要求处理循环引用）

答：核心逻辑分三步：

1. 判断类型：非引用类型直接返回，终止递归；
2. 处理循环引用：用 `WeakMap` 缓存已拷贝的对象，避免无限递归；
3. 递归拷贝：区分数组 / 对象，遍历自身属性并递归拷贝深层结构。
   （附上面试完整版代码，并说明 `WeakMap` 的优势）

#### （2）深拷贝中为什么用 WeakMap 而不是 Map？

答：

* `Map` 对键是强引用：若原对象被销毁，`Map` 中的键仍指向原对象，导致内存无法释放，引发内存泄漏；
* `WeakMap` 对键是弱引用：原对象销毁后，`WeakMap` 中的键会被垃圾回收自动清理，不占用额外内存，更适合处理循环引用的缓存。

### 3. 进阶拓展问

#### （1）如何拷贝 Map/Set/RegExp 等特殊类型？

答：在深拷贝函数中增加类型判断，针对性处理：

javascript

运行

```
function deepClone(target, map = new WeakMap()) {
  if (typeof target !== 'object' || target === null) return target;

  // 处理循环引用
  if (map.has(target)) return map.get(target);

  // 处理特殊类型
  if (target instanceof RegExp) {
    return new RegExp(target.source, target.flags);
  }
  if (target instanceof Date) {
    return new Date(target);
  }
  if (target instanceof Map) {
    const result = new Map();
    map.set(target, result);
    target.forEach((v, k) => {
      result.set(deepClone(k, map), deepClone(v, map));
    });
    return result;
  }
  if (target instanceof Set) {
    const result = new Set();
    map.set(target, result);
    target.forEach(v => {
      result.add(deepClone(v, map));
    });
    return result;
  }

  // 处理数组/普通对象
  const result = Array.isArray(target) ? [] : {};
  map.set(target, result);
  for (let key in target) {
    if (target.hasOwnProperty(key)) {
      result[key] = deepClone(target[key], map);
    }
  }
  return result;
}
```

#### （2）浅拷贝的应用场景？

答：

* 仅需修改表层属性，无需隔离深层数据（如表单初始值拷贝）；
* 性能优先场景：浅拷贝耗时远低于深拷贝，无需深层拷贝时优先使用；
* 扩展 / 修改对象部分属性（如 `const newObj = { ...obj, name: '新名称' }`）。

#### （3）深拷贝的性能优化思路？

答：

1. 提前终止递归：非引用类型直接返回，避免无效递归；
2. 缓存已拷贝对象：用 `WeakMap` 减少重复递归（如循环引用场景）；
3. 分片拷贝：大对象分批递归，避免阻塞主线程；
4. 优先使用原生 API：浏览器内置 `structuredClone` 支持循环引用、特殊类型，性能优于手写深拷贝。

## 三、面试回答技巧

1. **先易后难**：先讲定义→浅拷贝实现→深拷贝基础版→完整版（循环引用），逻辑层层递进；
2. **结合场景**：每个方法说明适用场景（如浅拷贝适合简单数据，深拷贝适合复杂嵌套数据）；
3. **主动拓展**：写完代码后主动提 `JSON.parse` 缺陷、`WeakMap` 优势，提升回答深度；
4. **避坑提醒**：强调 `Object.assign`、扩展运算符是浅拷贝，避免面试官追问时出错。

## 总结

1. 深浅拷贝核心：**是否复制引用类型的深层内存空间**；
2. 手写深拷贝关键：**递归 + 类型判断 + WeakMap 处理循环引用**；
3. 高频考点：`JSON.parse(JSON.stringify())` 缺陷、WeakMap 作用、特殊类型拷贝。
