# 二. 闭包（场景题：闭包的应用 + 内存泄漏问题）

### 一、面试答题核心框架（先定义，再场景，最后讲问题）

#### 1. 先讲闭包的核心定义（面试开场铺垫）

- **闭包的本质**：函数执行后，其执行上下文本应被销毁，但由于内部函数（子函数）引用了外部函数（父函数）作用域中的变量，导致父函数的作用域无法被垃圾回收（GC），这种「子函数能访问父函数作用域变量」的现象就是闭包。
- **核心特征**：
  1. 子函数嵌套在父函数内部；
  2. 子函数引用父函数的变量 / 参数；
  3. 子函数被外部调用（脱离父函数的执行上下文）。
- **简单示例（先写基础闭包，让面试官看到你理解本质）**：

javascript

运行

```javascript
function outer() {
  let a = 10; // 父函数变量
  // 子函数引用父变量，且被返回（外部可调用）
  return function inner() {
    console.log(a); // 访问父作用域变量
  };
}
const fn = outer();
fn(); // 10（outer执行完后，a未被销毁，因为inner引用了它）
```

#### 2. 闭包的实际应用场景（面试场景题核心，结合业务举例）

闭包的核心价值是「延长变量的生命周期」「私有化变量」，以下是面试高频场景：

##### 场景 1：私有化变量 / 模拟私有属性（经典场景）

JS 没有原生的 `private` 关键字，闭包可实现变量私有化，避免全局污染。

javascript

运行

```javascript
// 场景：实现一个计数器，只能通过方法修改计数，不能直接访问
function createCounter() {
  let count = 0; // 私有变量，外部无法直接修改
  // 闭包保留count的作用域
  return {
    increment: () => count++, // 增加计数
    decrement: () => count--, // 减少计数
    getCount: () => count, // 获取计数
  };
}

// 测试
const counter = createCounter();
counter.increment();
counter.increment();
console.log(counter.getCount()); // 2
console.log(counter.count); // undefined（无法直接访问私有变量）
```

##### 场景 2：防抖 / 节流函数（前端高频业务场景）

防抖和节流的核心是用闭包保存「定时器 ID」「上次执行时间」等变量，延长其生命周期。

javascript

运行

```javascript
// 场景：搜索框输入防抖（避免频繁请求接口）
function debounce(fn, delay) {
  let timer = null; // 闭包保存定时器ID
  // 返回的函数是闭包，引用了timer和delay
  return function (...args) {
    clearTimeout(timer); // 清除上一次的定时器
    timer = setTimeout(() => {
      fn.apply(this, args); //this指向使用这个函数的对象，思维发散this如果设置为null的话指向window,严格模式就指向null
    }, delay);
  };
}

// 测试：模拟搜索框输入回调
const search = (keyword) => {
  console.log(`请求搜索：${keyword}`);
};
const debounceSearch = debounce(search, 500);

// 模拟快速输入
debounceSearch("js");
debounceSearch("js闭包");
debounceSearch("js闭包应用");
// 最终只执行一次：请求搜索：js闭包应用（500ms无新输入后执行）
```

##### 场景 3：柯里化函数（参数复用）

javascript

运行

```javascript
// 场景：实现加法柯里化，复用第一个参数
function add(a) {
  // 闭包保存a的值
  return function (b) {
    return a + b;
  };
}

const add5 = add(5); // 复用参数5
console.log(add5(3)); // 8
console.log(add5(10)); // 15
```

##### 场景 4：循环绑定事件（经典面试场景题）

问题：循环给多个按钮绑定点击事件，直接绑定会导致所有按钮点击后都输出最后一个索引；用闭包可解决。

html

预览

```html
<button>按钮1</button>
<button>按钮2</button>
<button>按钮3</button>
<script>
  // 问题代码（无闭包）
  const btns = document.querySelectorAll("button");
  // for (var i = 0; i < btns.length; i++) {
  //   btns[i].onclick = function() {
  //     console.log(`点击了按钮${i+1}`); // 所有按钮都输出“点击了按钮4”
  //   };
  // }

  // 闭包解决：保留每次循环的i值
  for (var i = 0; i < btns.length; i++) {
    // 立即执行函数（IIFE）创建闭包，保存当前i值
    (function (index) {
      btns[index].onclick = function () {
        console.log(`点击了按钮${index + 1}`); // 正确输出对应按钮索引
      };
    })(i);
  }
</script>
```

## 闭包导致的内存泄漏问题（面试必问，重点讲 “原因 + 解决方案”）

##### （1）内存泄漏的原因

- 闭包会让父函数的作用域无法被垃圾回收，若闭包被长期引用（比如挂载到全局变量、DOM 元素），且作用域内有大量大体积变量（如 DOM 对象、大数组），会导致这些变量一直占用内存，无法释放，最终引发内存泄漏。
- 常见泄漏场景：
  javascript

  运行

  ```javascript
  // 示例：闭包引用DOM元素，且DOM删除后闭包仍存在
  function init() {
    const btn = document.getElementById("btn");
    btn.onclick = function () {
      console.log("点击按钮");
      // 闭包引用了btn，即使btn被移除，也无法被GC回收
    };
  }
  init();
  // 后续删除DOM，但闭包仍持有引用
  document.body.removeChild(document.getElementById("btn"));
  ```

##### （2）解决方案（面试重点，分点说，清晰易懂）

1. **及时解除引用**：不再使用闭包时，手动将闭包引用的变量设为 `null`，让垃圾回收器能回收内存。
   javascript

   运行

   ```javascript
   function init() {
     let btn = document.getElementById("btn");
     btn.onclick = function () {
       console.log("点击按钮");
     };
     // 解除引用：DOM删除后，手动置空
     btn = null;
   }
   ```
2. **避免闭包引用不必要的大变量**：只引用需要的变量，减少内存占用。
   javascript

   运行

   ```javascript
   function outer() {
     const bigData = new Array(1000000); // 大数组
     const smallData = 10; // 小变量
     // 优化前：闭包引用了bigData（即使不用）
     // return () => console.log(smallData);
     // 优化后：只保留需要的变量，避免引用大变量
     return (() => {
       const data = smallData;
       return () => console.log(data);
     })();
   }
   ```
3. **避免闭包挂载到全局**：全局变量的生命周期和页面一致，尽量将闭包挂载到局部作用域，或使用完后销毁。
   javascript

   运行

   ```javascript
   // 不好的写法：闭包挂载到全局，永远无法回收
   // window.fn = outer();
   // 好的写法：局部作用域使用，用完销毁
   (function () {
     const fn = outer();
     fn();
     fn = null; // 解除引用
   })();
   ```
4. **使用块级作用域（let/const）替代 var**：在循环等场景中，let/const 的块级作用域可减少闭包的使用，也避免变量泄漏。
   javascript

   运行

   ```javascript
   // 循环绑定事件：用let替代var，无需闭包
   for (let i = 0; i < btns.length; i++) {
     btns[i].onclick = function () {
       console.log(`点击了按钮${i + 1}`); // 正确输出
     };
   }
   ```

### 二、面试答题技巧

1. **先定义，再举例**：回答闭包时，先讲核心定义，再手写 1 个基础示例，让面试官快速 get 你的理解。
2. **场景题结合业务**：讲应用场景时，结合 “防抖节流、私有变量、循环绑定事件” 等前端实际业务，体现你能落地使用。
3. **内存泄漏讲 “原因 + 方案”**：不要只说 “闭包会导致内存泄漏”，要说明 “为什么会泄漏”“怎么解决”，体现深度。
4. **代码注释清晰**：手写代码时加关键注释（比如 `// 闭包保存定时器ID`），体现逻辑清晰。

### 总结

1. 闭包的核心：子函数引用父函数变量，且子函数被外部调用，导致父函数作用域无法被 GC 回收，本质是「作用域的保留」。
2. 闭包的核心应用：私有化变量、防抖 / 节流、柯里化、解决循环绑定事件的索引问题。
3. 闭包内存泄漏：原因是闭包长期引用大变量 / DOM 且未解除引用；解决方案是及时置空引用、避免引用不必要变量、减少全局挂载。

## this 指向问题


| 优先级 | 函数调用方式         | this 指向                              | 例子                                  |
| ------ | -------------------- | -------------------------------------- | ------------------------------------- |
| 1      | new 调用构造函数     | 新创建的实例对象                       | `new Student()`→ this=stu 实例        |
| 2      | call/apply/bind 调用 | 第一个参数指定的对象                   | `fn.call(obj)`→ this=obj              |
| 3      | 对象方法调用         | 调用方法的那个对象                     | `obj.fn()`→ this=obj                  |
| 4      | 普通函数调用（全局） | window（浏览器）/undefined（严格模式） | `fn()`→ this=window                   |
| 特殊   | 箭头函数             | 定义时外层普通函数的 this              | `obj.fn = () => {}`→ this = 外层 this |
