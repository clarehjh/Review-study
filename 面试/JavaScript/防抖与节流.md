# 前端面试之防抖节流（手写 + 业务场景）

## 一、核心概念（面试开篇必答，先定义再区分）

### 1. 防抖（Debounce）

* **定义**：触发事件后，延迟 `n` 毫秒再执行回调；如果在这 `n` 毫秒内又触发了事件，则重新计时。
* **核心**：「触发后等待，重复触发则重置等待时间」，最终只执行**最后一次**触发的回调。

### 2. 节流（Throttle）

* **定义**：触发事件后，立即执行回调；在接下来的 `n` 毫秒内，无论触发多少次事件，都不再执行。
* **核心**：「固定频率执行」，保证在单位时间内只执行**一次**回调。

### 3. 核心区别（一句话记死）

表格


| 特性     | 防抖                       | 节流                       |
| -------- | -------------------------- | -------------------------- |
| 执行时机 | 最后一次触发后延迟执行     | 触发后立即执行，后续限流   |
| 执行次数 | 多次触发可能只执行 1 次    | 多次触发按频率执行多次     |
| 核心目的 | 防止重复触发（如搜索输入） | 控制执行频率（如滚动加载） |

## 二、手写代码（面试核心，分基础版 + 进阶版）

### 1. 防抖函数（Debounce）

#### （1）基础版（立即执行版 / 非立即执行版二选一，优先写非立即）

javascript

运行

```
// 非立即执行版（常用：搜索框输入后延迟查询）
function debounce(fn, delay) {
  let timer = null; // 缓存定时器
  return function (...args) {
    // 每次触发，清空之前的定时器，重置计时
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args); // 绑定this，传递参数
    }, delay);
  };
}

// 立即执行版（可选：第一次触发立即执行，后续限流）
function debounceImmediate(fn, delay) {
  let timer = null;
  return function (...args) {
    const isFirst = !timer; // 判断是否是第一次触发
    clearTimeout(timer);
    timer = setTimeout(() => {
      timer = null; // 延迟结束后重置
    }, delay);
    if (isFirst) {
      fn.apply(this, args); // 第一次触发立即执行
    }
  };
}
```

#### （2）进阶版（支持取消 + 立即执行，面试加分）

javascript

运行

```
function debounce(fn, delay, immediate = false) {
  let timer = null;
  
  // 封装防抖后的函数
  const debounced = function (...args) {
    clearTimeout(timer);
    if (immediate) {
      // 立即执行逻辑：第一次触发执行，后续延迟内不执行
      const callNow = !timer;
      timer = setTimeout(() => {
        timer = null;
      }, delay);
      if (callNow) fn.apply(this, args);
    } else {
      // 非立即执行：最后一次触发后延迟执行
      timer = setTimeout(() => {
        fn.apply(this, args);
      }, delay);
    }
  };
  
  // 增加取消方法（业务场景：比如输入框清空后取消查询）
  debounced.cancel = function () {
    clearTimeout(timer);
    timer = null;
  };
  
  return debounced;
}
```

### 2. 节流函数（Throttle）

#### （1）时间戳版（常用：滚动加载、窗口 resize）

javascript

运行

```
function throttle(fn, interval) {
  let lastTime = 0; // 记录上一次执行时间
  return function (...args) {
    const now = Date.now();
    // 当当前时间 - 上一次执行时间 ≥ 间隔，才执行
    if (now - lastTime >= interval) {
      fn.apply(this, args);
      lastTime = now; // 更新上一次执行时间
    }
  };
}
```

#### （2）定时器版（面试可选，补充说明即可）

javascript

运行

```
function throttleTimer(fn, interval) {
  let timer = null;
  return function (...args) {
    if (!timer) { // 定时器不存在时才执行
      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null; // 执行后清空定时器
      }, interval);
    }
  };
}
```

## 三、业务场景应用（面试必答，结合代码示例）

### 1. 防抖的核心场景：搜索框输入联想

javascript

运行

```
// 业务场景：搜索框输入后，停止输入500ms再请求接口，避免频繁请求
const searchInput = document.getElementById('search-input');

// 原始查询函数
function searchHandler(value) {
  console.log('请求搜索接口：', value);
  // 真实场景：axios.get(`/api/search?keyword=${value}`)
}

// 绑定防抖函数（延迟500ms执行）
const debouncedSearch = debounce(searchHandler, 500);
searchInput.addEventListener('input', function (e) {
  debouncedSearch(e.target.value);
});

// 取消防抖（可选：比如点击取消按钮）
document.getElementById('cancel-btn').addEventListener('click', function () {
  debouncedSearch.cancel();
});
```

### 2. 节流的核心场景：滚动加载 / 窗口 resize

javascript

运行

```
// 业务场景1：滚动到底部加载更多，1秒内只触发一次
function loadMoreData() {
  console.log('加载更多数据');
  // 真实场景：请求分页接口，渲染数据
}
const throttledLoad = throttle(loadMoreData, 1000);
window.addEventListener('scroll', throttledLoad);

// 业务场景2：窗口resize时调整布局，500ms只执行一次
function resizeHandler() {
  console.log('调整页面布局');
}
const throttledResize = throttle(resizeHandler, 500);
window.addEventListener('resize', throttledResize);
```

## 四、面试高频问题（按提问概率排序）

### 1. 基础必问

#### （1）防抖和节流的区别是什么？

答：核心区别在「执行时机和次数」：

* 防抖：触发后延迟执行，重复触发则重置计时，最终只执行最后一次；
* 节流：触发后立即执行，单位时间内无论触发多少次，都只执行一次；
* 举例：搜索框用防抖（避免输入一次请求一次），滚动加载用节流（保证固定频率加载）。

#### （2）防抖函数中为什么要用 apply 绑定 this？

答：

* 防抖函数返回的是一个新函数，默认 `this` 会丢失（指向 window/undefined）；
* `apply(this, args)` 能把原事件的 `this`（比如搜索框 DOM 元素）和参数（比如输入值）传递给原函数，保证函数执行上下文正确。

#### （3）节流的时间戳版和定时器版有什么区别？

答：

* 时间戳版：触发后**立即执行**，结束时不会额外执行；
* 定时器版：触发后**延迟执行**，即使停止触发，最后一次仍会执行；
* 实际开发中优先用时间戳版（符合 “立即响应” 的业务预期）。

### 2. 手写代码必问

#### （1）手写防抖函数（要求支持取消、立即执行）

答：按 “缓存定时器→判断立即执行 / 非立即执行→绑定 this 和参数→增加取消方法” 的逻辑写（即进阶版代码），重点说明：

* `timer` 闭包缓存，保证多次触发共享同一个定时器；
* `immediate` 参数控制是否立即执行；
* `cancel` 方法用于手动取消防抖（比如输入框清空后取消查询）。

#### （2）手写节流函数（时间戳版）

答：核心逻辑：

1. 用 `lastTime` 记录上一次执行时间；
2. 每次触发时，判断当前时间与 `lastTime` 的差值是否≥间隔；
3. 满足条件则执行函数，并更新 `lastTime`。

### 3. 进阶拓展问

#### （1）防抖节流的应用场景还有哪些？

答：

* 防抖：
  1. 按钮点击（比如提交按钮，防止重复提交）；
  2. 输入框验证（比如手机号 / 邮箱验证，停止输入后再验证）；
  3. 浏览器返回按钮防抖（避免快速点击多次返回）；
* 节流：
  1. 鼠标移动 / 拖拽（比如拖拽元素时实时计算位置，限制频率）；
  2. 高频点击按钮（比如点赞按钮，1 秒内只能点一次）；
  3. 接口请求限流（比如秒杀按钮，防止一秒内多次请求）。

#### （2）如何实现一个既支持防抖又支持节流的函数？

答：封装通用函数，通过参数控制类型（面试可选，补充思路即可）：

javascript

运行

```
function debounceThrottle(fn, delay, type = 'debounce') {
  let timer = null;
  let lastTime = 0;
  return function (...args) {
    if (type === 'debounce') {
      // 防抖逻辑
      clearTimeout(timer);
      timer = setTimeout(() => fn.apply(this, args), delay);
    } else {
      // 节流逻辑
      const now = Date.now();
      if (now - lastTime >= delay) {
        fn.apply(this, args);
        lastTime = now;
      }
    }
  };
}
```

#### （3）原生 JS 中有没有类似防抖节流的 API？

答：

* `requestAnimationFrame` 可模拟节流（60 帧 / 秒，约 16ms 执行一次）；
* 新版浏览器支持 `EventTarget.addEventListener` 的 `passive` 参数，但本质不是防抖节流，只是优化滚动性能；
* 实际开发中仍以手写防抖节流为主，更灵活。

## 五、面试回答技巧

1. **先定义再举例**：先讲防抖节流的核心区别，再结合 “搜索框 / 滚动加载” 的业务场景，让回答更具象；
2. **手写代码分步骤**：先写基础版，再补充进阶功能（如取消、立即执行），体现思考过程；
3. **结合业务场景**：每个方法都说明 “为什么这个场景要用它”（比如搜索框用防抖是为了减少接口请求）；
4. **主动拓展**：写完代码后，主动提 `this` 绑定、取消方法、应用场景拓展，加分。

## 总结

1. 防抖核心：**最后一次触发后执行**，解决 “重复触发” 问题；
2. 节流核心：**固定频率执行**，解决 “触发太频繁” 问题；
3. 面试高频考点：手写防抖（支持取消）、手写节流（时间戳版）、场景落地、this 绑定。
