## 一、Vue2 响应式原理

### 1. 核心实现

Vue2 基于 `Object.defineProperty()` 实现响应式，核心流程：

* **数据劫持**：遍历 `data` 中的属性，通过 `Object.defineProperty()` 为每个属性添加 `getter/setter`。
* **依赖收集**：在 `getter` 中收集当前组件的 Watcher（依赖），存入 Dep（依赖管理器）。
* **派发更新**：在 `setter` 中触发 Dep 通知所有 Watcher 执行更新（重新渲染组件）。

### 2. 核心缺陷

表格


| 缺陷类型             | 具体表现                                                                                                     |
| -------------------- | ------------------------------------------------------------------------------------------------------------ |
| 数组响应式缺陷       | 无法监听数组的`push/pop/shift/unshift/splice/sort/reverse`等方法，也无法监听数组下标修改（如`arr[0] = 1`）。 |
| 对象新增属性缺陷     | 无法监听对象**新增 / 删除**属性（如`obj.newKey = 1`、`delete obj.key`）。                                    |
| 嵌套对象深度监听问题 | 需要递归遍历所有嵌套属性，初始化性能较差；若嵌套对象层级过深，劫持成本高。                                   |

### 3. 缺陷解决方案（Vue2 官方）

* 数组：重写数组原型方法（`push/pop` 等 7 个方法），在方法执行后触发更新。
* 对象 / 数组新增属性：使用 `Vue.set(target, key, value)` 或 `this.$set`；删除属性用 `Vue.delete`/`this.$delete`。

## 二、Vue3 响应式原理

### 1. 核心实现

Vue3 基于 `Proxy` + `Reflect` 实现响应式，核心流程：

* **代理拦截**：通过 `Proxy` 代理目标对象（支持对象 / 数组 / Map/Set 等），拦截 13 种操作（如 `get/set/deleteProperty`）。
* **依赖收集**：在 `get` 拦截器中收集依赖（Effect）。
* **派发更新**：在 `set/deleteProperty` 等拦截器中触发依赖重新执行。

### 2. Ref/Reactive 区别

表格


| 特性         | Reactive                           | Ref                                                         |
| ------------ | ---------------------------------- | ----------------------------------------------------------- |
| 作用对象     | 引用类型（对象 / 数组 / Map 等）   | 基本类型（字符串 / 数字 / 布尔）+ 引用类型                  |
| 底层实现     | Proxy 代理原对象                   | 基本类型：包裹为 {value: 数据}；引用类型：内部调用 reactive |
| 使用方式     | 直接访问属性（如`obj.name`）       | 需通过`.value`访问（模板中自动解包）                        |
| 响应式深度   | 深层响应式（递归代理所有嵌套属性） | 深层响应式（引用类型）                                      |
| 解构后响应式 | 解构后丢失响应式（需用 toRefs）    | 解构后仍保留响应式（`.value`不变）                          |

### 3. toRef/toRefs 作用

* **toRef**：
  * 作用：为响应式对象的单个属性创建一个 ref，保持与原对象的响应式关联。
  * 场景：需要将对象的某个属性单独传递，且希望修改该属性时同步更新原对象。
  * 示例：`const nameRef = toRef(obj, 'name')`。
* **toRefs**：
  * 作用：将响应式对象的所有属性转换为 ref，返回一个包含所有 ref 的普通对象，保持与原对象的响应式关联。
  * 场景：解构响应式对象时，避免丢失响应式（如 `const { name, age } = toRefs(obj)`）。
  * 注意：toRefs 仅处理对象已存在的属性，新增属性不会被转换。

## 三、面试高频题目 & 标准答案

### 1. 基础必问

#### 题目 1：说说 Vue2 响应式原理，为什么数组和对象新增属性不响应？

**标准答案**：

Vue2 基于 `Object.defineProperty()` 实现响应式，核心是遍历 data 中的属性，为每个属性添加 getter/setter：

* getter 中收集依赖（Watcher），setter 中触发依赖更新。
* 数组不响应的原因：`Object.defineProperty()` 只能监听属性的读写，无法监听数组方法（push/pop 等）和下标修改，Vue2 虽重写了 7 个数组方法，但未处理下标修改；
* 对象新增属性不响应的原因：初始化时仅对已有属性做了劫持，新增属性未被添加 getter/setter，自然无法触发更新；
* 解决：数组 / 对象新增属性用 `this.$set`，删除属性用 `this.$delete`。

#### 题目 2：Vue3 响应式原理和 Vue2 有什么区别？为什么 Vue3 用 Proxy 而不是 Object.defineProperty？

**标准答案**：

核心区别在于底层实现和能力：

1. 实现方式：
   * Vue2：`Object.defineProperty()` 劫持单个属性；
   * Vue3：`Proxy` 代理整个对象，配合 `Reflect` 完成拦截。
2. Proxy 的优势（解决 Vue2 缺陷）：
   * 支持监听数组方法 / 下标修改、对象新增 / 删除属性，无需手动调用 \$set；
   * 支持 Map/Set 等复杂数据类型，而 Object.defineProperty 仅支持普通对象；
   * 无需递归遍历嵌套对象（懒代理，访问时才劫持），初始化性能更好；
   * 能拦截更多操作（如 `delete`、`in` 等），功能更全面。
3. 其他区别：
   * Vue2 是 “属性级” 劫持，Vue3 是 “对象级” 代理；
   * Vue3 提供 Ref/Reactive 等 API，更灵活处理不同类型数据。

### 2. 进阶高频

#### 题目 3：Ref 和 Reactive 的区别？什么时候用 Ref，什么时候用 Reactive？

**标准答案**：

核心区别是作用对象和使用方式，选择依据如下：

1. 区别（核心）：
   * Reactive 适用于引用类型（对象 / 数组），直接访问属性，底层是 Proxy 代理；
   * Ref 适用于基本类型（字符串 / 数字），需通过 `.value` 访问，基本类型底层是包裹对象，引用类型内部调用 Reactive。
2. 使用场景：
   * 用 Ref：定义基本类型数据（如 `let count = ref(0)`）、需要解构的引用类型、跨组件传递的简单数据；
   * 用 Reactive：定义复杂对象 / 数组（如 `let user = reactive({ name: '张三' })`）、无需解构的引用类型。

#### 题目 4：toRef 和 toRefs 的作用？为什么需要它们？

**标准答案**：

toRef/toRefs 的核心作用是**保留响应式关联**，解决解构响应式对象丢失响应式的问题：

1. toRef：为响应式对象的单个属性创建 ref，修改该 ref 的 .value 会同步更新原对象；
   * 场景：仅需使用对象的某个属性，且希望保持响应式（如 `const name = toRef(user, 'name')`）。
2. toRefs：将响应式对象的所有属性转为 ref，返回普通对象，解构后仍能保持响应式；
   * 场景：解构响应式对象（如 `const { name, age } = toRefs(user)`），避免解构后修改属性不触发更新。
3. 为什么需要：
   * 直接解构 reactive 对象（如 `const { name } = user`），得到的是普通值，修改 name 不会触发更新；
   * toRef/toRefs 让解构后的属性仍与原对象关联，保持响应式。

#### 题目 5：Vue3 中 Ref 的 .value 为什么在模板中可以省略？

**标准答案**：

Vue3 模板编译阶段做了**自动解包**处理：

* 当模板中访问 Ref 数据时，Vue 会自动读取 `.value` 属性，无需手动写；
* 注意：在 `<script setup>` 中（非模板）必须写 `.value`，因为这是 JavaScript 运行时，Vue 无法自动解包；
* 特殊情况：Ref 作为对象属性时，模板中仍需 `.value`（如 `{{ obj.age.value }}`）。

## 四、核心总结

1. Vue2 响应式核心是 `Object.defineProperty()`，缺陷是无法监听数组方法 / 对象新增属性，需用 `$set/$delete` 解决；
2. Vue3 基于 `Proxy` 实现响应式，天然解决 Vue2 缺陷，通过 Ref/Reactive 区分数据类型，toRef/toRefs 解决解构丢失响应式问题；
3. 面试答题关键：先讲核心实现，再对比区别，最后结合使用场景，逻辑上从 “原理” 到 “问题” 再到 “解决方案”。
