### 1. 路由守卫（面试官高频追问）

#### 核心概念

路由守卫是 Vue Router 提供的导航钩子，用于控制路由跳转的权限、逻辑处理，分为**全局守卫、路由独享守卫、组件内守卫**。

#### 高频面试题 & 答案

##### （1）请说说 Vue Router 有哪些路由守卫？分别用在什么场景？

表格


| 守卫类型     | 常用钩子               | 核心使用场景                                                 |
| ------------ | ---------------------- | ------------------------------------------------------------ |
| 全局守卫     | `router.beforeEach`    | 登录鉴权、权限校验、页面加载进度条（如 NProgress）           |
|              | `router.afterEach`     | 页面标题修改、埋点统计、滚动条重置                           |
|              | `router.beforeResolve` | 解析异步路由后执行（如动态路由加载完成）                     |
| 路由独享守卫 | `beforeEnter`          | 单个路由的特殊权限校验（如管理员路由）                       |
| 组件内守卫   | `beforeRouteEnter`     | 进入组件前获取数据（不能访问`this`，需通过`next(vm => {})`） |
|              | `beforeRouteUpdate`    | 路由参数变化时更新数据（如 /id=1 → /id=2）                   |
|              | `beforeRouteLeave`     | 离开组件时确认（如表单未保存提示）                           |

##### （2）路由守卫中`next()`的参数有哪些用法？

javascript

运行

```
// 1. next()：放行，跳转到目标路由
// 2. next(false)：中断跳转，停留在当前页面
// 3. next('/login')：重定向到指定路由
// 4. next({ path: '/login', replace: true })：重定向且不保留历史记录
// 5. next(vm => {})：仅beforeRouteEnter可用，访问组件实例
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (to.path === '/login') return next()
  if (!token) return next('/login')
  next()
})
```

##### （3）如何解决路由守卫中异步操作导致的跳转问题？

javascript

运行

```
// 错误：异步操作后next()可能执行多次
router.beforeEach((to, from, next) => {
  // 正确：await异步操作后再执行next()
  async function checkAuth() {
    const hasAuth = await getAuthStatus()
    if (hasAuth) next()
    else next('/login')
  }
  checkAuth()
})
```

### 2. 动态路由（权限控制核心）

#### 核心概念

动态路由指根据用户权限动态生成路由表，常用于后台管理系统的权限控制，核心是`router.addRoute()`方法。

#### 高频面试题 & 答案

##### （1）如何实现基于权限的动态路由？

javascript

运行

```
// 1. 定义基础路由（无需权限）
const staticRoutes = [{ path: '/login', component: () => import('@/views/login.vue') }]
// 2. 定义权限路由模板
const asyncRoutes = [
  { path: '/admin', component: () => import('@/views/admin.vue'), meta: { role: ['admin'] } },
  { path: '/editor', component: () => import('@/views/editor.vue'), meta: { role: ['admin', 'editor'] } }
]
// 3. 登录后根据用户角色筛选路由
export const initDynamicRouter = async () => {
  const userRole = await getUserRole() // 从接口获取用户角色
  const accessibleRoutes = asyncRoutes.filter(route => {
    if (!route.meta?.role) return true
    return route.meta.role.includes(userRole)
  })
  // 4. 动态添加路由
  accessibleRoutes.forEach(route => router.addRoute(route))
}
// 4. 在路由守卫中触发
router.beforeEach(async (to, from, next) => {
  if (!router.hasRoute(to.name)) {
    await initDynamicRouter()
    return next({ ...to, replace: true }) // 重新触发路由跳转
  }
  next()
})
```

##### （2）动态路由刷新后 404 怎么解决？

* 原因：刷新后动态添加的路由丢失，路由匹配失败跳转到 404
* 解决方案：
  1. 刷新后重新请求用户权限，再次初始化动态路由；
  2. 将 404 路由放在动态路由添加完成后注册（避免优先匹配 404）：
     javascript

     运行

     ```
     // 错误：提前注册404导致动态路由被覆盖
     const router = createRouter({ routes: [...staticRoutes, { path: '*', component: NotFound }] })
     // 正确：动态路由添加后再注册404
     export const add404Route = () => {
       router.addRoute({ path: '/:pathMatch(.*)*', component: () => import('@/views/404.vue') })
     }
     ```

### 3. 路由懒加载（性能优化必问）

#### 核心概念

路由懒加载（按需加载）通过`import()`语法拆分代码包，减少首屏加载体积，提升页面加载速度。

#### 高频面试题 & 答案

##### （1）如何实现 Vue 路由懒加载？

javascript

运行

```
// 方式1：基础懒加载
const Home = () => import('@/views/Home.vue')

// 方式2：懒加载+代码分割命名（便于打包分析）
const About = () => import(/* webpackChunkName: "about" */ '@/views/About.vue')

// 方式3：批量导入（适合路由较多场景）
const loadView = (view) => {
  return () => import(`@/views/${view}.vue`)
}
const routes = [
  { path: '/home', component: loadView('Home') },
  { path: '/about', component: loadView('About') }
]
```

##### （2）路由懒加载的原理是什么？

* 核心：利用 ES6 的`import()`动态导入语法，返回 Promise 对象；
* 打包时 Webpack 会将每个懒加载的路由组件拆分为独立的 chunk 文件；
* 路由跳转时才加载对应的 chunk 文件，实现按需加载。

### 4. hash/history 模式区别（底层原理必问）

#### 核心对比表

表格


| 对比维度   | hash 模式                                              | history 模式                                                                 |
| ---------- | ------------------------------------------------------ | ---------------------------------------------------------------------------- |
| URL 格式   | `http://xxx.com/#/home`                                | `http://xxx.com/home`                                                        |
| 底层原理   | 基于 URL 的 hash 值（#），通过`hashchange`事件监听变化 | 基于 HTML5 History API（`pushState`/`replaceState`），通过`popstate`事件监听 |
| 刷新 404   | 不会 404（hash 部分不会发送到服务器）                  | 刷新会 404（需后端配置）                                                     |
| SEO 友好性 | 差（搜索引擎忽略 hash 部分）                           | 好（URL 更规范）                                                             |
| 兼容性     | 兼容所有浏览器（包括 IE）                              | 兼容 IE10+                                                                   |
| 配置成本   | 无需后端配置                                           | 后端需配置：所有路由指向 index.html                                          |

#### 高频面试题 & 答案

##### （1）history 模式刷新 404 怎么解决？

* 后端配置示例（Nginx）：
  nginx

  ```
  server {
    listen 80;
    server_name xxx.com;
    root /usr/share/nginx/html;
    index index.html;

    # 核心配置：所有路由指向index.html
    location / {
      try_files $uri $uri/ /index.html;
    }
  }
  ```
* 后端配置示例（Apache）：
  apache

  ```
  <IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.html [L]
  </IfModule>
  ```

##### （2）项目中如何选择 hash/history 模式？

* 小型项目 / 无需 SEO：hash 模式（配置简单）；
* 中大型项目 / 需要 SEO/URL 美观：history 模式（需后端配合）；
* 兼容低版本浏览器：hash 模式。

## 二、Vuex/Pinia 核心考点（15K + 必掌握）

### 1. 核心概念（基础必背）

#### Vuex 核心概念

表格


| 概念     | 作用                                 | 核心语法                                                     |
| -------- | ------------------------------------ | ------------------------------------------------------------ |
| State    | 存储全局状态（单一状态树）           | `this.$store.state.xxx`/`mapState`                           |
| Getter   | 计算属性（基于 State 派生）          | `this.$store.getters.xxx`/`mapGetters`                       |
| Mutation | 修改 State 的唯一方式（同步）        | `this.$store.commit('mutationName', payload)`/`mapMutations` |
| Action   | 处理异步逻辑（可提交 Mutation）      | `this.$store.dispatch('actionName', payload)`/`mapActions`   |
| Module   | 拆分模块化状态（解决单一状态树臃肿） | `const moduleA = { state, mutations, actions, getters }`     |

#### Pinia 核心概念

* **Store**：替代 Vuex 的 Module，每个 Store 是独立的；
* **State**：存储状态（函数返回对象，支持响应式）；
* **Getters**：计算属性（支持自动缓存）；
* **Actions**：处理同步 / 异步逻辑（无需 Mutation）；
* **Plugins**：扩展 Pinia 功能（如持久化）。

### 2. Pinia 优势（对比 Vuex 必问）

#### 核心优势表

表格


| 对比维度      | Vuex（3.x/4.x）                          | Pinia                                       |
| ------------- | ---------------------------------------- | ------------------------------------------- |
| 语法复杂度    | 繁琐（需 Mutation、Module 命名空间）     | 简洁（无 Mutation，原生 TS 支持）           |
| TS 支持       | 差（需手动类型声明）                     | 优秀（原生 TS 支持，类型推导）              |
| 模块化        | 需手动配置命名空间（`namespaced: true`） | 天然模块化（每个 Store 独立）               |
| 代码体积      | 较大                                     | 更小（约 1KB）                              |
| 异步处理      | 需通过 Action 提交 Mutation              | Actions 可直接修改 State（同步 / 异步均可） |
| DevTools 支持 | 支持                                     | 支持（且更完善，可追踪 Store 变化）         |
| 组合式 API    | 适配差                                   | 完美适配 Vue3 组合式 API                    |

#### 高频面试题 & 答案

##### （1）为什么推荐 Vue3 项目使用 Pinia 替代 Vuex？

1. 语法更简洁：去掉 Mutation，Actions 可直接修改 State，减少冗余代码；
2. 原生 TS 支持：无需额外配置，类型推导更完善，开发体验更好；
3. 天然模块化：无需手动配置命名空间，每个 Store 独立，便于维护；
4. 体积更小：Pinia 体积仅 1KB 左右，比 Vuex 更轻量；
5. 适配组合式 API：完美兼容 Vue3 的`setup`语法，代码更统一。

##### （2）Pinia 中如何实现 Store 之间的通信？

javascript

运行

```
// store/user.js
import { defineStore } from 'pinia'
export const useUserStore = defineStore('user', {
  state: () => ({ name: '张三' })
})

// store/cart.js
import { defineStore } from 'pinia'
import { useUserStore } from './user'

export const useCartStore = defineStore('cart', {
  actions: {
    addCart(item) {
      // 方式1：直接引入并使用其他Store
      const userStore = useUserStore()
      console.log('当前用户：', userStore.name)
      this.items.push(item)
    }
  }
})
```

### 3. 持久化方案（项目实战必问）

#### （1）Vuex 持久化（vuex-persistedstate）

javascript

运行

```
// 安装：npm i vuex-persistedstate
import { createStore } from 'vuex'
import createPersistedState from 'vuex-persistedstate'

const store = createStore({
  modules: {
    user: {
      state: () => ({ token: '', name: '' }),
      mutations: { /* ... */ }
    }
  },
  plugins: [
    createPersistedState({
      key: 'vuex-store', // 存储的key名
      storage: localStorage, // 存储方式（localStorage/sessionStorage）
      paths: ['user.token'] // 只持久化user模块的token字段
    })
  ]
})
```

#### （2）Pinia 持久化（pinia-plugin-persistedstate）

javascript

运行

```
// 安装：npm i pinia-plugin-persistedstate
import { createPinia } from 'pinia'
import persist from 'pinia-plugin-persistedstate'

const pinia = createPinia()
pinia.use(persist)

// 定义Store时配置持久化
export const useUserStore = defineStore('user', {
  state: () => ({ token: '', name: '' }),
  persist: {
    key: 'user-store',
    storage: localStorage,
    paths: ['token'] // 只持久化token
  }
})
```

#### 高频面试题 & 答案

##### （1）手动实现 Pinia 持久化（不用插件）

javascript

运行

```
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('user_token') || '',
    name: ''
  }),
  actions: {
    setToken(token) {
      this.token = token
      // 同步到localStorage
      localStorage.setItem('user_token', token)
    },
    clearToken() {
      this.token = ''
      localStorage.removeItem('user_token')
    }
  }
})
```

##### （2）持久化方案的注意事项？

1. 敏感数据（如 token）建议加密存储（如 base64 / 加密算法）；
2. 避免持久化大量数据（影响性能）；
3. 区分`localStorage`（永久存储）和`sessionStorage`（会话级存储）；
4. 多标签页同步：监听`storage`事件更新状态。
