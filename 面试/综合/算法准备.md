### 一、15k + 前端面试中，算法考察的核心范围

前端面试的算法题有非常明确的 “高频边界”，不会考动态规划、贪心、图论等复杂算法，主要集中在这几类：

1. **基础手写题（和 JS 核心结合，必考）**

   这类题其实你在复习 JS 核心时已经接触，本质是 “算法思维 + JS API 运用”，比如：
   - 防抖 / 节流（核心：定时器 + 条件判断）
   - 深浅拷贝（核心：递归 + 类型判断）
   - Promise.all/race（核心：异步控制 + 数组遍历）
   - 数组去重 / 扁平化 / 排序（核心：数组 API + 循环）
   - 实现数组的 map/filter/reduce（核心：遍历 + 回调函数）

2. **简单逻辑题（考察基础思维，高频）**

   这类题代码量少，重点看你能否理清逻辑，比如：
   - 两数之和（LeetCode 1，前端面试最常考的入门题）
   - 反转字符串 / 数字（LeetCode 344/7，纯字符串 / 数字操作）
   - 判断回文字符串（LeetCode 125，基础循环 + 对比）
   - 扁平数据转树形结构（前端业务高频，比如菜单数据处理）
   - 树形结构转扁平数据（和上面互逆，业务常用）

3. **复杂度分析（了解即可，不用深入）**

   不用背复杂公式，只需要能说出 “时间复杂度 O (n)、空间复杂度 O (1)” 这类基础结论，比如：
   - 遍历数组的时间复杂度是 O (n)
   - 递归深拷贝的时间复杂度是 O (n)（n 为对象属性总数）
   - 用 Set 去重的时间复杂度是 O (n)

### 二、算法复习方案（1 周内搞定，轻量化）

你可以在第 2 周 / 第 3 周抽出每天 1-2 小时，按这个顺序复习，不用贪多：

#### 第一步：先搞定 “业务型算法题”（前端特有，性价比最高）

这类题和前端业务强相关，面试考到的概率最高，也是你能快速掌握的：

1. **数组 / 对象处理（1 天）**
   - 必练题：
     ✅ 数组去重（至少 2 种方法：Set、遍历 + includes）
     ✅ 数组扁平化（至少 2 种方法：递归、flat (Infinity)、reduce）
     ✅ 扁平数据转树形（比如：`[{id:1, pid:0}, {id:2, pid:1}]` → 树形结构）
   - 示例代码（扁平转树形，前端业务高频）：
     javascript

     运行

     ```
     function arrayToTree(arr, pid = 0) {
       // 时间复杂度O(n)，空间复杂度O(n)
       const result = [];
       // 先遍历一次，用对象存id和元素的映射，避免重复遍历
       const map = {};
       arr.forEach(item => {
         map[item.id] = item;
         item.children = []; // 初始化children
       });
       // 二次遍历，根据pid挂载子节点
       arr.forEach(item => {
         if (item.pid === pid) {
           result.push(item);
         } else {
           if (map[item.pid]) {
             map[item.pid].children.push(item);
           }
         }
       });
       return result;
     }
     // 测试用例
     const data = [
       { id: 1, pid: 0, name: '首页' },
       { id: 2, pid: 1, name: '产品' },
       { id: 3, pid: 2, name: '产品详情' },
       { id: 4, pid: 0, name: '我的' }
     ];
     console.log(arrayToTree(data)); // 输出树形结构
     ```

2. **JS API 手写（1 天）**
   - 必练题：
     ✅ 手写数组的 map 方法
     ✅ 手写数组的 reduce 方法
     ✅ 手写 Promise.all（核心：计数器 + Promise.resolve）
   - 核心：理解这些 API 的 “输入 - 处理 - 输出” 逻辑，不用追求 100% 和原生一致，重点体现核心逻辑。

#### 第二步：搞定 “入门级算法题”（LeetCode 简单题，保底）

不用刷太多，选 5-8 道前端高频的简单题，每题写 1 遍，理解逻辑即可：

表格

| LeetCode 题号 | 题目名称       | 核心考点                  | 难度 |
| ------------- | -------------- | ------------------------- | ---- |
| 1             | 两数之和       | 数组遍历 + Map / 对象缓存 | 简单 |
| 7             | 整数反转       | 数字 / 字符串操作         | 简单 |
| 13            | 罗马数字转整数 | 映射表 + 遍历             | 简单 |

- 示例（两数之和，前端面试最常考）：
  javascript

  运行

  ```
  // 方法1：暴力遍历（时间O(n²)，空间O(1)）
  function twoSum(nums, target) {
    for (let i = 0; i < nums.length; i++) {
      for (let j = i + 1; j < nums.length; j++) {
        if (nums[i] + nums[j] === target) {
          return [i, j];
        }
      }
    }
    return [];
  }
  // 方法2：Map缓存（时间O(n)，空间O(n)，更优）
  function twoSum(nums, target) {
    const map = new Map();
    for (let i = 0; i < nums.length; i++) {
      const complement = target - nums[i];
      if (map.has(complement)) {
        return [map.get(complement), i];
      }
      map.set(nums[i], i);
    }
    return [];
  }
  ```

#### 第三步：背 “复杂度分析话术”（应对面试追问）

不用深入理解算法复杂度的推导，只需要记住高频题的结论，比如：

- 暴力遍历的时间复杂度：O (n²)
- 单次遍历 / Map 缓存的时间复杂度：O (n)
- 递归深拷贝的时间复杂度：O (n)（n 为属性总数）
- 用 Set/Map 的空间复杂度：O (n)
- 原地修改数组（比如双指针）的空间复杂度：O (1)

面试时如果被问 “这个解法的复杂度是多少？”，直接说结论即可，不用推导，前端面试官不会深究。

### 三、避坑提醒（算法较弱的情况下）

1. **不要贪多求难**：比如动态规划、贪心、图论等，15k + 前端岗位几乎不考，不用浪费时间；
2. **不要只看不动手**：每道题一定要自己写一遍（哪怕抄答案后改一遍），面试时是 “手写”，看会≠写会；
3. **面试时坦诚面对**：如果被问到不会的算法题，不要慌，可以说 “这个题我暂时没写过，但我理解核心思路是 XXX，我可以试着一步步写”，然后边说边写，前端面试官更看重 “思维过程” 而非 “结果 100% 正确”；
4. **结合你的项目**：如果算法题和你的项目相关（比如 “小程序中如何处理大量订单数据的排序？”），可以结合项目说 “我在羽球助手中处理订单列表时，用了 XXX 方法（比如数组 sort + 防抖），避免了性能问题”，弱化纯算法的不足，强化业务落地能力。

### 总结

1. **必要性**：15k + 前端岗位需要复习算法，但只聚焦 “前端高频简单题”，不用深入复杂算法；
2. **核心范围**：业务型算法（扁平转树形、数组处理）+ JS 手写题（防抖、Promise.all）+ LeetCode 简单题（两数之和、反转字符串）；
3. **复习策略**：1 周时间，每天 1-2 小时，动手写代码 + 记复杂度结论，优先保证 “能写、能讲”，而非 “最优解”。

你的核心优势是 Vue/Uniapp/ 工程化，算法只需要达到 “能应对基础面试题” 的水平即可，不用追求和后端 / 算法岗一样的深度，把更多时间留给 React 和 AI 前端应用的复习，性价比更高。
