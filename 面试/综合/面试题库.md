## 一、Vue3 核心原理与特性

### 1. Vue3 响应式原理是什么？

**答：** Vue3 使用 Proxy 代替了 Vue2 的 Object.defineProperty，实现了对对象的深层次监听。Proxy 可以直接监听数组和对象的新增、删除属性，性能更优，代码更简洁。

### 2. Composition API 和 Options API 有什么区别？

**答：** Composition API 更灵活，逻辑复用性更强，适合大型项目和复杂组件。Options API 结构清晰，适合小型项目和初学者。Composition API 通过 `setup` 函数组织逻辑，Options API 通过 data、methods、computed 等分块组织。

### 3. Vue3 中的 Teleport、Suspense、Fragment 是什么？

**答：**

- **Teleport**：可以将子组件渲染到 DOM 的任意位置。
- **Suspense**：用于异步组件的加载等待，配合异步 setup 使用。
- **Fragment**：支持组件返回多个根节点，减少无意义的 DOM 层级。

### 4. Vue3 的 diff 算法优化了哪些点？

**答：** Vue3 使用了静态提升（Hoist Static）、Block Tree 优化，减少了不必要的 diff 和 patch，提高了渲染性能。

### 5. Vue3 中的 ref 和 reactive 区别？

**答：**

- **ref** 用于基本类型和对象，返回一个带有 .value 的响应式引用。
- **reactive** 只能用于对象，返回对象本身的响应式代理。

## 二、前端基础

### 1. 事件委托是什么？

**答：** 事件委托是利用事件冒泡，将子元素的事件处理委托到父元素上，减少事件绑定数量，提高性能。

### 2. 盒模型有哪些？如何切换？

**答：** 标准盒模型（content-box）和 IE 盒模型（border-box）。通过 `box-sizing` 属性切换。

### 3. 闭包的作用和常见场景？

**答：** 闭包可以访问函数外部的变量，常用于封装私有变量、实现柯里化、定时器等。

### 4. 防抖和节流的区别？

**答：** 防抖（debounce）是事件触发后延迟执行，若期间再次触发则重新计时。节流（throttle）是规定时间内只执行一次。

## 五、浏览器原理与安全

### 1. 浏览器渲染流程？

**答：** 解析 HTML → 构建 DOM 树 → 解析 CSS → 构建 CSSOM → 合成渲染树 → 布局（Reflow）→ 绘制（Repaint）。

### 2. XSS 和 CSRF 的防护？

**答：**

- XSS：输入输出过滤、使用 CSP、转义特殊字符。
- CSRF：验证码、Referer 校验、Token 校验。

### 3. 跨域的常见解决方案？

**答：** CORS、JSONP、代理服务器、postMessage、nginx 反向代理等。

## 八、Vue3 进阶与生态

### 1. Vue3 中的自定义指令如何实现？

**答：** 通过 `directive` 选项或 `app.directive` 注册，生命周期钩子有 created、beforeMount、mounted、beforeUpdate、updated、beforeUnmount、unmounted。

### 2. Vue3 如何实现跨组件通信？

**答：**

- props / emit
- provide / inject
- mitt 等事件总线
- Vuex/Pinia 状态管理
- ref/父子组件引用

### 3. 说说 Pinia 和 Vuex 的区别？

**答：** Pinia 是 Vue3 官方推荐的新一代状态管理库，API 更简洁，支持组合式 API，类型推导更好，模块自动拆分，无需 mutation。

### 4. Vue3 如何做服务端渲染（SSR）？

**答：** 使用 `@vue/server-renderer`，配合 Node.js 服务端渲染，提升首屏速度和 SEO。

### 5. Vue3 组件库生态有哪些？

**答：** Element Plus、Naive UI、Vant、Ant Design Vue 等。

## 九、JavaScript 深入原理

### 1. 事件循环（Event Loop）机制？

**答：** JS 是单线程，事件循环分为宏任务（如 setTimeout）和微任务（如 Promise.then），微任务优先于宏任务执行。

### 2. 原型链和继承？

**答：** 每个对象有 **proto** 指向其原型，构成原型链。继承可通过原型链、ES6 class extends、Object.create 等实现。

### 3. this 的指向？

**答：** this 取决于调用方式：普通函数、对象方法、构造函数、箭头函数（继承外层 this）、call/apply/bind 显式绑定。

### 4. 深拷贝和浅拷贝？

**答：** 浅拷贝只复制一层引用，深拷贝递归复制所有层级。常用 JSON.parse(JSON.stringify(obj))，但有缺陷，推荐用 lodash 的 cloneDeep。

### 5. ES6+ 新特性？

**答：** let/const、箭头函数、模板字符串、解构赋值、Promise、async/await、Set/Map、Symbol、可选链、空值合并、模块化等。

## 十、CSS 进阶与布局

### 1. Flex 和 Grid 布局区别？

**答：** Flex 适合一维布局（行或列），Grid 适合二维布局（行和列同时控制）。

### 2. BFC（块级格式化上下文）是什么？

**答：** BFC 是一种布局环境，常用于清除浮动、避免 margin 重叠。触发方式有 overflow、float、display:inline-block/table-cell 等。

### 3. 响应式设计常用方案？

**答：** 媒体查询（Media Query）、rem/em、flex、vw/vh、百分比布局、移动端适配（viewport、flexible.js）。

### 4. 如何实现一个三栏自适应布局？

**答：** 可用 Flex、Grid、float+margin、圣杯/双飞翼布局等。

### 5. CSS 性能优化？

**答：** 精简选择器、减少嵌套、避免通配符、合并样式、使用硬件加速（transform、opacity）、合理使用动画。

## 十一、网络与安全

### 1. HTTP2/HTTP3 有哪些新特性？

**答：** 多路复用、头部压缩、服务器推送、0-RTT（HTTP3）、基于 UDP（HTTP3）。

### 2. 常见的 HTTP 状态码？

**答：** 200（成功）、301/302（重定向）、304（缓存）、400（请求错误）、401/403（未授权/禁止）、404（未找到）、500（服务器错误）。

### 3. 前端缓存策略？

**答：** 强缓存（Expires/Cache-Control）、协商缓存（Last-Modified/Etag）、Service Worker 离线缓存。

### 4. WebSocket 和 HTTP 的区别？

**答：** WebSocket 是持久化、全双工通信协议，适合实时应用。HTTP 是无状态、单向请求响应。

### 5. 前端如何防止 SQL 注入？

**答：** 前端应避免拼接 SQL，所有数据交由后端参数化处理，前端主要做输入校验和转义。

## 十二、性能优化细节

### 1. 如何减少重排（Reflow）和重绘（Repaint）？

**答：** 批量操作 DOM、使用 DocumentFragment、避免逐条修改样式、合并动画帧、使用 transform 替代 top/left。

### 2. 懒加载和预加载的区别？

**答：** 懒加载是按需加载资源，预加载是提前加载未来可能用到的资源。

### 3. 如何监控前端性能？

**答：** 使用 Performance API、Lighthouse、Web Vitals、埋点上报、Sentry、阿里云前端监控等。

### 4. 如何优化大数据量渲染？

**答：** 虚拟列表（virtual scroll）、分页、分片渲染、Web Worker。
